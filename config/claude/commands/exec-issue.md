# 依頼内容

Issue の内容を確認し、タスクを遂行すること。
タスクはワークフローに従い進めること。

# ワークフロー

1. ghを使用してissue $1を参照、対応内容を把握する
   Parent issueがある場合は内容を把握。
   疑問や確認事項がある場合、各自判断して先に進める。
   判断内容や理由は後述する実装計画に記載する。
2. docs/work_logsに`[日付]*[対応内容]`のディレクトリを作成
   　例: `20260315_show_required_informations`
3. 対応に必要な調査を詳細に実施
   調査はチームを作成し、sonnetで実行。
4. 調査結果をもとに、2.のディレクトリに調査資料(research.md)を作成する
   `mo`コマンドでresearch.mdを開く
5. 調査資料を元に実装計画を立て、2.のディレクトリに実装計画(plan.md)を作成する
   `mo`コマンドでplan.mdを開く
6. 実装着手
7. 実装完了後、ユーザーの承認を得る

# 注意点

ステップ6以降でも実装や計画に変更が入った場合は適宜plan.mdを更新すること。
