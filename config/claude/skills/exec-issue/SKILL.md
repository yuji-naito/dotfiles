# Exec Issue

## 依頼内容

Issue の内容を確認し、タスクを遂行すること。
タスクはワークフローに従い進めること。

## ワークフロー

### 1.  ghを使用してissue $1を参照、対応内容を把握する
Parent issueがある場合は内容を把握。
疑問や確認事項がある場合、各自判断して先に進める。
判断内容や理由は後述する実装計画に記載する。

### 2. ドキュメントディレクトリを作成
docs/work_logsに`[日付]*[対応内容]`のディレクトリを作成する。
例: `20260315_show_required_informations`

### 3. 対応前調査
対応に必要な調査を詳細に実施。
調査はチームを作成し、sonnetで実行。
調査結果は2.のディレクトリに調査資料(research.md)としてまとめる。

### 4. 実装計画
調査資料を元に実装計画を立てる。
2.のディレクトリに実装計画(plan.md)を作成する。

### 5. ドキュメントを表示
調査資料、実装計画を`mo`コマンドで開く。
ユーザー確認用だが、承認なしで進める。

### 6. 実装
調査資料、実装計画を元に実装を行う。

### 7. コミット、PR作成
ユーザーの承認を得て、`/commit`コマンドを実行してコミット、`/create-pr`コマンドでPRを作成する。


## 注意点

調査資料、実装計画は都度更新し、コードと乖離が生じないようにすること。
