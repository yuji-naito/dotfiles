# 依頼内容

Issue の内容を確認し、タスクを遂行すること。
タスクはワークフローに従い進めること。

# ワークフロー

1. ghを使用してissue $1を参照、対応内容を把握する
   Parent issueがある場合は内容を把握。
   疑問、確認点がある場合はユーザーに質問を行い、その内容を元にissueを更新する。
2. docs/work_logsに`[日付]*[対応内容]`のディレクトリを作成
   　例: `20260315_show_required_informations`
3. 対応に必要な調査を詳細に実施
   調査はチームを作成し、sonnetで実行。
4. 調査結果をもとに、2.のディレクトリに調査資料(research.md)を作成する
5. ユーザーに確認を依頼し、承認を得たら次のステップに進む
6. 2.のディレクトリに実装計画(plan.md)を作成
7. ユーザーに確認を依頼し、承認を得たら次のステップに進む
8. 実装着手

# 注意点

ステップ6以降でも計画に変更が入った場合は適宜plan.mdを更新すること。
